Research Insights: What Makes Browser Games Go Viral

The .io Formula (Agar.io → 100M+ players, ZombsRoyale.io → 120M+ players):
One-click start → One core mechanic → Real-time leaderboard → Permadeath + instant restart → Emergent social dynamics
Compulsion Loop
Anticipation → Action → Reward → Repeat. Short cycle times trigger dopamine. Players who die should be back in-game within 1 second.
Easy to Learn, Hard to Master
Agar.io is "move mouse, eat dots" — but mastery requires deep strategy. Skill ceiling keeps players engaged long after they learn basics.
Viral by Design
Slither.io unlocked cosmetic skins by sharing the game link. Zero marketing budget, every player became a marketer.
Juice = Feel
Screen shake, particles, squash-and-stretch, sound effects. High visual value for low performance cost. Polish amplifies strong mechanics.

Recommended Execution Timeline

Sprint 1 (Week 1–2): 1.8, 5.1, 5.3, 5.4 — Canvas renderer + proper game loop foundation
Sprint 2 (Week 3): 1.1–1.6 — Game feel / juice (screen shake, particles, SFX)
Sprint 3 (Week 4): 2.1, 2.2, 2.7 — Addictive loop (instant restart, ghost line, streaks)
Sprint 4 (Week 5–6): 3.1–3.3, 3.5, 3.7 — Viral/social features (share, challenge, skins)
Sprint 5 (Week 7–8): 2.3–2.6, 4.1–4.3 — Progression system + visual polish
Sprint 6 (Week 9–10): 4.4–4.7, 5.5–5.7 — Advanced gameplay + performance tuning
Sprint 7+ (Month 3+): 6.x, 7.x — Multiplayer + monetization
Priority #1: Migrate to Canvas 2D rendering (1.8 / 5.1). The current DOM-based approach creates/destroys up to 200 div elements — this will bottleneck everything else. After that, add juice (screen shake, particles, sound effects). These two changes alone will transform how the game feels.

Priority #2: Instant restart (2.1) + one-click share (3.1). These are the two highest-ROI features for retention and virality. Every extra second between death and the next round loses players. Every share is free marketing.
